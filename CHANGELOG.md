## 1.0.0-dev.1 (2025-10-27)


### ⚠ BREAKING CHANGES

* Removed old CI pipeline, replaced with semantic-release workflow
* **sync:** Complete refactoring from API-first to filesystem-first approach

This is a fundamental architectural change that addresses the root cause of
missing service badges and duplicate entries. The filesystem is now the
source of truth, not external APIs.

Architecture Changes:
- NEW: pkg/filesystem/scanner.go - Scans real files, detects hardlinks via inode
- NEW: pkg/filesystem/enricher.go - Enriches files with service metadata
- NEW: internal/service/filesystem_sync_service.go - Filesystem-first sync implementation

Scanner Features:
- Walks filesystem for real media files (.mkv, .mp4, .avi, etc.)
- Detects hardlinks using Unix inode matching (syscall.Stat_t)
- Groups hardlinked files by inode
- Prioritizes paths: Library > Download > Alphabetical
- Infers media type (movie/series) from path patterns
- Configurable: root paths, extensions, min size, skip hidden

Enricher Features:
- Enriches scanned files with Radarr/Sonarr/Jellyfin/qBittorrent metadata
- Service-specific enrichment methods (EnrichWithRadarr, etc.)
- Fuzzy path matching: exact → primary → basename normalization
- Applies metadata: title, quality, poster, tags, torrent info

Database Model Updates:
- Added filesystem fields: FileInode, FileModTime, IsHardlink, HardlinkPaths, PrimaryPath
- Added service flags: InRadarr, InSonarr, InJellyfin, InJellyseerr, InQBittorrent
- FilePath now has uniqueIndex constraint

Configuration:
- Added FilesystemConfig with scan paths and options
- Defaults: /BibliotecaMultimedia root, library paths prioritized
- Video extensions: .mkv, .mp4, .avi, .m4v, .ts, .m2ts
- Min size: 100MB default

Benefits:
- One physical file = one database record (no duplicates)
- Perfect hardlink detection (same inode = same file)
- Multiple services can reference the same file
- Solves issue: "items in Jellyfin and Sonarr only showing Sonarr badge"
- Source of truth: filesystem, not inconsistent API responses

Flow:
OLD: API calls → Try to match paths → Database (duplicates, matching errors)
NEW: Filesystem scan → Detect hardlinks → Enrich with APIs → Database

Related Issues:
- Fixes missing qBittorrent badges (path mismatch /Descargas vs /Peliculas)
- Fixes items appearing multiple times (hardlinks seen as different files)
- Fixes missing service badges (same file, multiple service associations)
